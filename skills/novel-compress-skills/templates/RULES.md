# 提炼与删除规则 + Voice Design 对话格式

> 阶段一从 templates/ 复制到工作目录，填自定义区后确认。

## 一、输出格式

```
角色名(声音听起来什么样)：对话内容
旁白：叙述内容
```

Voice Design 描述的是**声音的物理属性**，不是角色的心理状态。
详见 `references/emotion_tags.md`——只一行：描述必须回答"这声音听起来什么样"。

❌ 禁止的抽象词：不甘心、无奈、挑衅、戏谑、意味深长、崩溃抓狂、若有所思...

✅ 只写声学属性：音高（拔高/沉下去）、语速（一字一顿/连珠炮/拖长尾音）、音量（压低/气声/突然拔高）、气息（叹气开头/气息不稳）、颤抖（发颤/劈叉）、发紧（喉咙像被掐住）、断裂（一句话分成几截/哽咽）

## 二、提炼清单（正面）

### 核心人物
- 首次出场描写 + 名场面台词 → 从原文复制
- 口头禅、标志动作

### 世界观
- 专有名词不可用泛指替代
- 数值必须精确

### 伏笔与钩子
- 每章结尾悬念必须保留

## 三、删除清单

| 类别 | 处置 |
|------|------|
| 重复心理 | 保留首次，后续只留一句 |
| 冗余过渡 | 删移动过程，保留到达后行动 |
| 注水对话 | 保留最有力一句 |
| 路人背景 | 保留名字+身份 |
| 环境碎碎念 | 保留1-2句关键意象 |

## 四、禁用词

"综上所述""由此可见""值得一提的是""毋庸置疑""不得不说""赋能""闭环""抓手""基石""保驾护航"、排比套话、鸡汤结尾

## 五、Bible 回写（强制，不询问）

写完压缩稿后**立刻**回写 bible，不要问、不要等、不要跳过。

- 新人物：写 characters.md（名字+层级+一句话+来源章）
- 新设定：写 settings.md
- 新时间线节点：写 timeline.md
- 新钩子：写 throughline.md
- 层级：新人物一律标"路人"，确认重要后升级为"弧级"或"核心"

- 不写 bible 文件，返回 delta.json
- 主流程合并 delta → 写入 bible
