# 方法论参考（来自深度调研，含 46 条来源）

> 完整报告见 `基于Codex目标导向模式的百万字小说压缩改编方法论.md`。此处为浓缩版，供技能使用时快速对齐原理。

## 一、可行性根因：分治
- 百万字 ≈ 150–200 万 token；GPT-5.4 标准窗口 272K、实验 1M；即便 1M 也仅够放"精简版"，整本直喂物理上不可能。
- 长文"中间迷失"（Lost in the Middle, Liu et al. 2024）：答案放中段召回率约 53.8%，低于闭卷基线 56.1%——小说主体恰好落在中段。
- 缺乏外部记忆 → 人设崩塌、伏笔失忆（社区共识：朴素方法约 15–25 章开始漂移）。
- Map-Reduce 摘要已被 OpenAI 书籍摘要、清华 LLM×MapReduce 实证可行；gpt-4o-mini 处理 5 万 token 约 $0.01，单文档压缩率 90%+。

## 二、核心流水线三件套
1. **分块**：以语义单元（章节结尾+空行+下章标题）为断点，带约 1200 token 重叠区；摘要任务块大小约 1k–4k token。
2. **剧情圣经**：把"事实"从"散文"解耦；每章抽取 delta 回写，下一章只喂"大纲+压缩状态"，绝不回喂原文。
3. **递归/顺序摘要**：顺序敏感、叙事性强 → 用 Refine（连贯但慢）；纯并行 Map-Reduce 会丢跨块连接。

## 三、Codex /goal 长跑
- 机制：规划→行动→验证→复盘续作循环；GPT-5.3-Codex 实测连跑约 25 小时/13M token。
- 三文件信任架构：`GOAL.md`（完成标准可运行命令）+ `VERIFY.md`（每条需求映射到检查命令）+ `LOG.md`（执行日志）。
- 三类失败模式：规格漂移、静默质量退化、验证作秀。
- 规避：独立 verifier 子 Agent（只读沙箱、高推理）；大任务拆 2–4 小时小 goal；设 token 软停止；主观删留保留人工签收。

## 四、提示词四要素（RULES.md 落地）
- 删什么：水章判定=删掉不影响后续即剔除。
- 留什么：主线+主角+关键钩子+独门意象/口头禅。
- 写成什么样：去 AI 味禁用清单（排比/模板词/黑话）。
- 怎么算完：30%–50% 压缩比 + 钩子无遗漏 + 不新增剧情。

## 五、风险与对冲
- **泛化失真**：生成式摘要在 26%–73% 案例过度概括（Peters & Chin-Yee, Royal Society Open Science 2025）；抽取式幻觉率近 0。对策：关键情节抽取式保底 + bible 事实 diff。
- **静默截断**：输出 token 上限无报错截断（早期 GPT-4o 4,096；现更高）。对策：verify.py 查字数；自然边界切分。
- **合规越界**：自动删除"垃圾/恶心章节"需人工签收（写 exclude.txt + 记录理由），满足可审计与申诉。

## 四条关键发现
1. 分治是可行性根因；回写纪律比工具选择更重要。
2. /goal 可靠性来自外置契约（三文件+独立 verifier），非模型本身。
3. 提示词即契约；分层规则文件替代超长 prompt（超长 prompt 截断率可达 38%）。
4. 失真/截断/越界三道风险必须机制兜底，压缩比不是越高越好。
